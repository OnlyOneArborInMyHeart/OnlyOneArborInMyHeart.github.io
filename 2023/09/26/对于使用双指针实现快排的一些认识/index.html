对于使用双指针实现快排的一些认识

No tag
Publish Date:   2023-09-26 
首先，我们对于快速排序要有一个基本的认识,快速排序实际上是基于冒牌排序的优化，但是对于冒泡排序的O(n^2)的时间复杂度，快速排序可以达到O(nlogn)的时间复杂度，

可以说，对于一个十分巨大的数据集来说，快速排序的优势，无疑是巨大的。
那么，计算机界的前辈们是怎么通过冒泡排序来想到快速排序的呢?以下是笔者的一些浅薄认知，如有错误，欢迎指出，首先我们对于一个正常使用冒泡排序的数组，我们都是从末尾开始进行冒泡排序
而被指针指向的那个数在经历过一遍排序过后，它的前一个数和后一个数会呈现怎么样的特点呢？如果这个冒泡排序的目的是产生一个递增数组的话，那么当前指向的这个数在经历过一遍
循环之后，它的前一个数一定小于它，而它的后一个数一定大于它，这是非常好理解的。也是基于这一点，我们想，如果我们在一个正常数组中随机挑选一个数，然后让比它小的数都在
它的左边，而比它大的数，自然就在它的右边。然后我们对这个数的左边和右边两个数组分别再进行这样一次处理，那么我们最终对于数组中的每一个数，都会有和第一个数相同的特点
而这，不就是一个有序数组吗？！
以上仅是笔者根据自己的认知用自然语言来描述的快速排序的思路，但是再计算机科学中，我们其实是使用了分治加双指针的思路来实现的（当然也有其他的实现方式，不过笔者认为
这种方式写出来更加简洁）
以下给出C++的双指针+递归实现快速排序:
#include
using namespace std;
const int N = 1e6 + 10;
int n ;
int p[N];
void quick_sort(int p[] , int l , int r){
if(l >= r) return;
int i = l - 1;
int j = r + 1;
int x = p[l];
while(i < j){
do i++; while(p[i] < x);
do j–; while(p[j] > x);
if(i < j) swap(p[i] , p[j]);
}
quick_sort(p , l , j);
quick_sort(p , j + 1 , r);
}
int main(){
scanf(“%d” , &n);
for(int i = 0 ; i < n ; i++){
scanf(“%d” , &p[i]);
}
quick_sort(p , 0 , n - 1)；
for(int i = 0 ; i < n ; i++){
printf(“%d” , p[i]);
}
return 0;
}
这已经是笔者遍查各种blog能找出的最简实现了，还忘海涵。
下面我们来谈谈其中的特殊点:
总共有两处:
1.当关键字选取的是最左边的数字的时候，我们下方的quick_sort()中的分界点只能是j,另外一个则是j + 1,
其实这个我们用一个特例即可说明，就是数组: 1 , 2
刚开始时，数组的i指针指向0 , j指针指向1,但是由于我们取的是左边，最后i和j会重合在左边界，但是在下一次递归中
如果我们使用i 和 i - 1的组合的话，那么则会分成两个新的边界即(0 , -1) 和 (0 , 1)，且不谈第一个区间的合理性，第二个
区间就是我们的原区间，这样是会造成死循环的！
事实上这是因为因为我们选取左边界，而左边界上的数在第一次循环的时候是不会移动的，所以当任何已经排序好的数组，使用错误情况时
i和j都会重合到下标为0的位置上，从而产生一个和原来一样的数组，引起死循环。
2.当笔者自己学习的时候曾问自己这么一个问题，就是我们实现了一个数组中从某个下标开始左边全小于和右边全大于的情况，但是这个被选中的数
本身却没有去到它该去的位置上，难道不会产生死循环的问题吗，但是单纯的抽象并不好理解，但是只要调试一组实际数就会发现，最后还是会被排好序
这是因为，在数字没有去往它应该去的位置时，再下一次再次选中这个数的时候会再次引起更小范围的部分排序，直到我们对于每一个数都如此处理为止，
那么也说明了一个问题：真正认识一种算法的有效方法就是带入实际数据后自己调试一遍，然后每一步都要关注他们的具体作用，它们又是怎么处理难处理的
问题的呢，然后在学习之后再手敲一遍。
事实上，快速排序的实现方法不止用双指针一种，而且双指针的写法也不止这一种，大家只需选择自己写的得心应手的即可，其实像填坑法和双指针中交换关键字的
方法背后的实现原理也十分精彩，鼓励大家也可以去了解背后的实现原理。
